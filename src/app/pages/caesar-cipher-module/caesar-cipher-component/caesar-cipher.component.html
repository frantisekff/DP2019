Description of attack
To break a Caesar cipher we decide to use a brute force attack with statistical analysis. We can find all combinations of keys because Caesar cipher has a small number of combinations. We are working with the telegraphic English alphabet (26 letters), so it is relatively quick to find the correct key/shift.

Frequencies and Index of coincidence
First we need to find out the length of key/shift. For this we must understand the frequencies of text and index of coincidence.

We calculate the frequency of letters by counting the occurrence of every letter in the text, then we divide it with the length of text. After that we convert it to a percentage.

Index coincidence (IC) is based on the relative frequencies of letters. It reflects the accuracy that two texts (choose randomly) will be the same. With IC we can identify the language of the text.
